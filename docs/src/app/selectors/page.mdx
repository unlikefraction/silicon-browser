# Selectors

## Refs (recommended)

Refs provide deterministic element selection from snapshots. Best for AI agents.

```bash
# 1. Get snapshot with refs
silicon-browser snapshot
# Output:
# - heading "Example Domain" [ref=e1] [level=1]
# - button "Submit" [ref=e2]
# - textbox "Email" [ref=e3]
# - link "Learn more" [ref=e4]

# 2. Use refs to interact
silicon-browser click @e2                   # Click the button
silicon-browser fill @e3 "test@example.com" # Fill the textbox
silicon-browser get text @e1                # Get heading text
silicon-browser hover @e4                   # Hover the link
```

### Why refs?

- **Deterministic** - Ref points to exact element from snapshot
- **Fast** - No DOM re-query needed
- **AI-friendly** - LLMs can reliably parse and use refs

## CSS selectors

```bash
silicon-browser click "#id"
silicon-browser click ".class"
silicon-browser click "div > button"
silicon-browser click "[data-testid='submit']"
```

## Text & XPath

```bash
silicon-browser click "text=Submit"
silicon-browser click "xpath=//button[@type='submit']"
```

## Semantic locators

Find elements by role, label, or other semantic properties:

```bash
silicon-browser find role button click --name "Submit"
silicon-browser find label "Email" fill "test@test.com"
silicon-browser find placeholder "Search..." fill "query"
silicon-browser find testid "submit-btn" click
```
