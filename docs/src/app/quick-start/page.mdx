# Quick Start

## Core workflow

Every browser automation follows this pattern:

```bash
# 1. Navigate
silicon-browser open example.com

# 2. Snapshot to get element refs
silicon-browser snapshot -i
# Output:
# @e1 [heading] "Example Domain"
# @e2 [link] "More information..."

# 3. Interact using refs
silicon-browser click @e2

# 4. Re-snapshot after page changes
silicon-browser snapshot -i
```

## Common commands

```bash
silicon-browser open example.com
silicon-browser snapshot -i                # Get interactive elements with refs
silicon-browser click @e2                  # Click by ref
silicon-browser fill @e3 "test@example.com" # Fill input by ref
silicon-browser get text @e1               # Get text content
silicon-browser screenshot                 # Save to temp directory
silicon-browser screenshot page.png        # Save to specific path
silicon-browser close
```

## Traditional selectors

CSS selectors and semantic locators also supported:

```bash
silicon-browser click "#submit"
silicon-browser fill "#email" "test@example.com"
silicon-browser find role button click --name "Submit"
```

## Headed mode

Show browser window for debugging:

```bash
silicon-browser open example.com --headed
```

## Wait for content

```bash
silicon-browser wait @e1                   # Wait for element
silicon-browser wait --load networkidle    # Wait for network idle
silicon-browser wait --url "**/dashboard"  # Wait for URL pattern
silicon-browser wait 2000                  # Wait milliseconds
```

## Command chaining

Chain commands with `&&` in a single shell call. The browser persists via a background daemon, so chaining is safe and efficient:

```bash
# Open, wait, and snapshot in one call
silicon-browser open example.com && silicon-browser wait --load networkidle && silicon-browser snapshot -i

# Chain multiple interactions
silicon-browser fill @e1 "user@example.com" && silicon-browser fill @e2 "pass" && silicon-browser click @e3

# Navigate and capture
silicon-browser open example.com && silicon-browser wait --load networkidle && silicon-browser screenshot page.png
```

Use `&&` when you don't need intermediate output. Run commands separately when you need to parse output first (e.g., snapshot to discover refs before interacting).

## JSON output

For programmatic parsing in scripts:

```bash
silicon-browser snapshot --json
silicon-browser get text @e1 --json
```

Note: The default text output is more compact and preferred for AI agents.
