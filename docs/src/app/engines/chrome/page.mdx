# Chrome

Chrome (and Chromium) is the default browser engine. Silicon Browser discovers, launches, and manages the browser process automatically via the Chrome DevTools Protocol (CDP).

## Binary Discovery

When no `--executable-path` is provided, Silicon Browser searches for a browser in this order:

<table>
  <thead>
    <tr><th>Platform</th><th>Locations checked</th></tr>
  </thead>
  <tbody>
    <tr>
      <td>macOS</td>
      <td>
        CloakBrowser cache (<code>~/.cloakbrowser/</code> or <code>CLOAKBROWSER_CACHE_DIR</code>),
        <code>/Applications/Google Chrome.app</code>,
        <code>/Applications/Google Chrome Canary.app</code>,
        <code>/Applications/Chromium.app</code>,
        <code>/Applications/Brave Browser.app</code>,
        Puppeteer cache (<code>~/.cache/puppeteer/chrome/</code> or <code>PUPPETEER_CACHE_DIR</code>),
        Chrome for Testing cache
      </td>
    </tr>
    <tr>
      <td>Linux</td>
      <td>
        CloakBrowser cache (<code>~/.cloakbrowser/</code> or <code>CLOAKBROWSER_CACHE_DIR</code>),
        <code>google-chrome</code>,
        <code>google-chrome-stable</code>,
        <code>chromium-browser</code>,
        <code>chromium</code> in PATH,
        Puppeteer cache (<code>~/.cache/puppeteer/chrome/</code> or <code>PUPPETEER_CACHE_DIR</code>),
        Chrome for Testing cache
      </td>
    </tr>
    <tr>
      <td>Windows</td>
      <td>
        CloakBrowser cache (<code>%USERPROFILE%\.cloakbrowser</code> or <code>CLOAKBROWSER_CACHE_DIR</code>),
        <code>%LOCALAPPDATA%\Google\Chrome\Application\chrome.exe</code>,
        <code>C:\Program Files\Google\Chrome\Application\chrome.exe</code>,
        <code>C:\Program Files (x86)\...\chrome.exe</code>
      </td>
    </tr>
  </tbody>
</table>

If no local browser is found, run `silicon-browser install` to download CloakBrowser plus Chrome for Testing. Silicon Browser prefers CloakBrowser and automatically retries once with a fresh implicit default profile if a stale `silicon` profile prevents CloakBrowser from completing CDP bootstrap.

## Usage

Chrome is the default engine -- no `--engine` flag is needed:

```bash
silicon-browser open example.com
```

To be explicit:

```bash
silicon-browser --engine chrome open example.com
```

## Custom Binary

Point to any Chromium-based browser with `--executable-path`:

```bash
silicon-browser --executable-path /path/to/chromium open example.com
```

Or via environment variable:

```bash
export SILICON_BROWSER_EXECUTABLE_PATH=/path/to/chromium
silicon-browser open example.com
```

## Containers and CI

In Docker, CI runners, or other sandboxed environments, Chrome's user namespace sandbox may need to be disabled:

```bash
silicon-browser --args "--no-sandbox" open example.com
```

Silicon Browser automatically adds `--no-sandbox` when it detects a container environment (Docker, Podman, running as root).
