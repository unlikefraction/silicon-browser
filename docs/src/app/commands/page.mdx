# Commands

## Core

```bash
silicon-browser open <url>              # Navigate (aliases: goto, navigate)
silicon-browser click <sel>             # Click element (--new-tab to open in new tab)
silicon-browser dblclick <sel>          # Double-click
silicon-browser fill <sel> <text>       # Clear and fill
silicon-browser type <sel> <text>       # Type into element
silicon-browser press <key>             # Press key (Enter, Tab, Control+a) (alias: key)
silicon-browser keyboard type <text>    # Type at current focus (no selector needed)
silicon-browser keyboard inserttext <text>  # Insert text without key events
silicon-browser keydown <key>           # Hold key down
silicon-browser keyup <key>             # Release key
silicon-browser hover <sel>             # Hover element
silicon-browser focus <sel>             # Focus element
silicon-browser select <sel> <val>      # Select dropdown option
silicon-browser check <sel>             # Check checkbox
silicon-browser uncheck <sel>           # Uncheck checkbox
silicon-browser scroll <dir> [px]       # Scroll (up/down/left/right, --selector <sel>)
silicon-browser scrollintoview <sel>    # Scroll element into view
silicon-browser drag <src> <dst>        # Drag and drop
silicon-browser upload <sel> <files>    # Upload files
silicon-browser screenshot [path]       # Screenshot (--full for full page)
silicon-browser screenshot --annotate   # Annotated screenshot with numbered element labels
silicon-browser screenshot --screenshot-dir ./shots    # Save to custom directory
silicon-browser screenshot --screenshot-format jpeg --screenshot-quality 80
silicon-browser pdf <path>              # Save page as PDF
silicon-browser snapshot                # Accessibility tree with refs
silicon-browser eval <js>               # Run JavaScript
silicon-browser connect <port|url>      # Connect to browser via CDP
silicon-browser stream enable [--port <port>]  # Start runtime WebSocket streaming
silicon-browser stream status           # Show runtime streaming state and bound port
silicon-browser stream disable          # Stop runtime WebSocket streaming
silicon-browser close                   # Close browser (aliases: quit, exit)
silicon-browser close --all             # Close all active sessions
```

## Find elements

Semantic locators with actions (`click`, `fill`, `type`, `hover`, `focus`, `check`, `uncheck`, `text`):

```bash
silicon-browser find role <role> <action> [value]
silicon-browser find text <text> <action>
silicon-browser find label <label> <action> [value]
silicon-browser find placeholder <ph> <action> [value]
silicon-browser find alt <text> <action>
silicon-browser find title <text> <action>
silicon-browser find testid <id> <action> [value]
silicon-browser find first <sel> <action> [value]
silicon-browser find last <sel> <action> [value]
silicon-browser find nth <n> <sel> <action> [value]
```

Options:

- `--name <name>` -- filter role by accessible name
- `--exact` -- require exact text match

Examples:

```bash
silicon-browser find role button click --name "Submit"
silicon-browser find label "Email" fill "test@test.com"
silicon-browser find alt "Logo" click
silicon-browser find first ".item" click
silicon-browser find last ".item" text
silicon-browser find nth 2 ".card" hover
```

## Downloads

```bash
silicon-browser download <sel> <path>   # Click element to trigger download
silicon-browser wait --download [path]  # Wait for any download to complete
```

Use `--download-path <dir>` (or `SILICON_BROWSER_DOWNLOAD_PATH` env) to set a default download directory. Without it, downloads go to a temporary directory that is deleted when the browser closes.

### Iframe support

Iframes are detected automatically during snapshots. `Iframe` nodes are resolved and their content is
inlined beneath the iframe element in the snapshot output. Refs assigned to elements inside iframes carry
frame context, so `click`, `fill`, and other interactions work without manually switching frames.

```bash
silicon-browser snapshot -i
# @e3 [Iframe] "payment-frame"
#   @e4 [input] "Card number"
#   @e5 [button] "Pay"

# Interact directly using refs — no frame switch needed
silicon-browser fill @e4 "4111111111111111"
silicon-browser click @e5

# Or switch frame context for scoped snapshots
silicon-browser frame @e3
silicon-browser snapshot -i             # Only elements inside that iframe
silicon-browser frame main              # Return to main frame
```

The `frame` command accepts element refs (`@e3`), CSS selectors (`"#my-iframe"`), or frame name/URL.

## Dialogs

```bash
silicon-browser dialog accept [text]    # Accept dialog (with optional prompt text)
silicon-browser dialog dismiss          # Dismiss dialog
silicon-browser dialog status           # Check if a dialog is currently open
```

By default, `alert` and `beforeunload` dialogs are automatically accepted so they never block the agent. `confirm` and `prompt` dialogs still require explicit handling. Use `--no-auto-dialog` (or `SILICON_BROWSER_NO_AUTO_DIALOG=1`) to disable automatic handling.

When a JavaScript dialog (`alert`, `confirm`, `prompt`) is pending, all command responses include a `warning` field with the dialog type and message.

## Auth vault

```bash
silicon-browser auth save <name> [opts]    # Save auth profile
silicon-browser auth login <name>          # Login using saved credentials
silicon-browser auth list                  # List saved profiles (names and URLs only)
silicon-browser auth show <name>           # Show profile metadata (no passwords)
silicon-browser auth delete <name>         # Delete a saved profile
```

Save options:

- `--url <url>` -- login page URL (required)
- `--username <user>` -- username (required)
- `--password <pass>` -- password (required unless `--password-stdin`)
- `--password-stdin` -- read password from stdin (recommended to avoid shell history exposure)
- `--username-selector <sel>` -- custom CSS selector for username field
- `--password-selector <sel>` -- custom CSS selector for password field
- `--submit-selector <sel>` -- custom CSS selector for submit button

`auth login` navigates with `load` and then waits for the username/password/submit selectors to appear before interacting. This improves reliability on SPA login pages where fields render after initial page load.

```bash
echo "pass" | silicon-browser auth save github --url https://github.com/login --username user --password-stdin
silicon-browser auth login github
silicon-browser auth list
```

## Confirmation

When `--confirm-actions` is set, certain action categories return a `confirmation_required` response instead of executing immediately. Use `confirm` or `deny` to approve or reject the action.

```bash
silicon-browser confirm <confirmation-id>  # Approve a pending action
silicon-browser deny <confirmation-id>     # Deny a pending action
```

Pending confirmations auto-deny after 60 seconds.

```bash
silicon-browser --confirm-actions eval,download eval "document.title"
# Returns confirmation_required with ID
silicon-browser confirm c_8f3a1234
```

## Chat

Use natural language to control the browser via AI. The `chat` command translates instructions into silicon-browser commands, executes them, and streams the AI response. Requires `AI_GATEWAY_API_KEY` to be set.

```bash
silicon-browser chat "open google.com and search for cats"     # Single-shot instruction
silicon-browser chat                                           # Interactive REPL (type quit to exit)
echo "summarize this page" | silicon-browser chat              # Piped input
silicon-browser -q chat "summarize this page"                  # Quiet: text only, no tool calls shown
silicon-browser -v chat "fill in the login form"               # Verbose: show commands and their output
silicon-browser --model openai/gpt-4o chat "take a screenshot" # Override the default AI model
silicon-browser --json chat "open example.com"                 # Structured JSON output
```

Chat-specific options:

```bash
--model <name>           # AI model (or AI_GATEWAY_MODEL env, default: anthropic/claude-sonnet-4.6)
-v, --verbose            # Show tool commands and their raw output
-q, --quiet              # Show only the AI text response (hide tool calls)
```

## Batch execution

Execute multiple commands in a single invocation. Commands can be passed as quoted arguments or piped as JSON via stdin.

```bash
# Argument mode: each quoted argument is a full command
silicon-browser batch "open https://example.com" "snapshot -i" "screenshot"

# With --bail to stop on first error
silicon-browser batch --bail "open https://example.com" "click @e1" "screenshot"

# Stdin mode: pipe commands as JSON
echo '[
  ["open", "https://example.com"],
  ["snapshot", "-i"],
  ["click", "@e1"],
  ["screenshot", "result.png"]
]' | silicon-browser batch --json
```

<table>
  <thead>
    <tr><th>Option</th><th>Description</th></tr>
  </thead>
  <tbody>
    <tr><td><code>--bail</code></td><td>Stop on first error (default: continue all commands)</td></tr>
    <tr><td><code>--json</code></td><td>Output results as a JSON array</td></tr>
  </tbody>
</table>

## Command chaining

Chain commands with `&&` in a single shell invocation. The browser persists via a background daemon, so chaining works naturally and is more efficient than separate calls:

```bash
silicon-browser open example.com && silicon-browser wait --load networkidle && silicon-browser snapshot -i
silicon-browser fill @e1 "user@example.com" && silicon-browser fill @e2 "pass" && silicon-browser click @e3
silicon-browser open example.com && silicon-browser wait --load networkidle && silicon-browser screenshot page.png
```

Use `&&` when you don't need to read intermediate output. Run commands separately when you need to parse output first (e.g., snapshot to discover refs, then interact with those refs).

## Local files

Open local files (PDFs, HTML) using `file://` URLs:

```bash
silicon-browser --allow-file-access open file:///path/to/document.pdf
silicon-browser --allow-file-access open file:///path/to/page.html
silicon-browser screenshot output.png
```

The `--allow-file-access` flag enables JavaScript to access other local files. Chromium only.
